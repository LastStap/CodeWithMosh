# Щоденні завдання Java Backend Developer

Цей файл деталізує програму з `CourseProgramm.md`.

## Правила ведення файлу

- За один раз до файлу додаються завдання лише для **одного навчального дня**.
- Наступний день додається тільки після окремого запиту.
- Нові завдання мають спиратися на відповідний день із `CourseProgramm.md`.
- Перед додаванням наступного дня потрібно врахувати результат попереднього:
  виконані завдання, труднощі, помилки та теми для повторення.
- Не потрібно переходити до наступного дня, якщо не виконані критерії
  завершення поточного дня.
- Орієнтовна тривалість дня — **5 годин 30 хвилин чистого навчального часу**.
- Після кожних 50-60 хвилин роботи потрібно зробити перерву 10-15 хвилин.

# День 1. Середовище, перша Java-програма та Git

## Мета дня

Підготувати й перевірити робоче середовище, зрозуміти базову структуру
Java-програми, навчитися запускати її з IntelliJ IDEA та термінала, а також
перевірити базове розуміння Git.

Наприкінці дня ти повинен:

- знати встановлені версії Java, Maven і Git;
- розуміти призначення JDK, JVM, JRE, source code та bytecode;
- самостійно написати клас `HelloDeveloper`;
- скомпілювати й запустити програму з IntelliJ IDEA;
- скомпілювати й запустити програму з термінала;
- знайти створений `.class`-файл;
- розуміти базову структуру наявного Maven-проєкту;
- переглянути зміни через Git і підготувати змістовний commit.

## 3. Керована практика — 60 хвилин

### Завдання 3.1. Перевір інструменти

У терміналі, з кореня проєкту, виконай команди по черзі:

```bash
pwd
java -version
javac -version
mvn -version
git --version
git status
```

Запиши в `learning-journal.md`:

- поточну директорію;
- версію Java;
- версію `javac`;
- версію Maven;
- версію Git;

яку показали `java -version` та `javac -version`. Сьогодні не змінюй
`pom.xml` лише заради збігу версій. Якщо версії несумісні, запиши це як
окреме спостереження.

### Завдання 3.2. Досліди Maven-проєкт

Знайди й відкрий:

- `pom.xml`;
- `src/main/java`;
- `src/test/java`;
- `target`.

Потім знайди в `pom.xml`:

- `groupId`;
- `artifactId`;
- `version`;
- Java compiler source і target;
- test dependency.

Запиши їхні поточні значення. Поки що нічого в `pom.xml` не змінюй.

### Завдання 3.3. Повтори першу програму за курсом

В IntelliJ IDEA створи файл:


```text
Hello, Java Developer!
Day 1: Environment is ready.
```

Не копіюй готовий клас. Самостійно напиши:

- package declaration;
- class declaration;
- метод `main`;
- два виклики `System.out.println`.

Запусти клас через IntelliJ IDEA.

### Завдання 3.4. Зміни поведінку

1. Заміни текст першого рядка на привітання зі своїм ім'ям.

3. Навмисно видали одну крапку з комою.

## 4. Самостійне завдання — 120 хвилин

Цей блок виконай без перегляду рішення курсу. Документацію та власний
конспект використовувати можна. AI або готове рішення — лише після
щонайменше 30 хвилин самостійної спроби.

### Завдання 4.1. Розшир `HelloDeveloper`

Програма повинна вивести структуровану картку:

```text
================================
JAVA BACKEND LEARNING PROGRAM
================================
Student: <твоє ім'я>
Target role: Junior Java Backend Developer
Target country: Germany
Current day: 1 of 84
Today's topic: Java environment and execution
================================
```

Вимоги:

- кожен логічний рядок виводиться окремою командою;
- текст не повинен починатися або закінчуватися випадковими пробілами;
- результат має легко читатися в консолі;
- назва класу має точно збігатися з назвою файла;
- package має відповідати директорії.

### Завдання 4.2. Запусти програму з термінала через Maven

З кореня проєкту виконай:

```bash
mvn compile
```

Після успішної компіляції знайди:

```text
target/classes/daniil/dumshenko/HelloDeveloper.class
```

Запусти програму:

```bash
java -cp target/classes daniil.dumshenko.previousLessons.HelloDeveloper
```

Запиши:

- що створила команда `mvn compile`;
- навіщо використовується `-cp target/classes`;
- чому в команді запуску вказано повне ім'я
  `daniil.dumshenko.previousLessons.HelloDeveloper`.

### Завдання 4.3. Скомпілюй без Maven

Створи тимчасову директорію для результату компіляції:

```bash
mkdir -p /tmp/codewithmosh-day1
```

Скомпілюй клас:

```bash
javac -d /tmp/codewithmosh-day1 src/main/java/daniil/dumshenko/HelloDeveloper.java
```

Запусти його:

```bash
java -cp /tmp/codewithmosh-day1 daniil.dumshenko.previousLessons.HelloDeveloper
```

Порівняй:

- запуск через IntelliJ IDEA;
- компіляцію через Maven;
- ручну компіляцію через `javac`.

У конспекті напиши по 1-2 речення про те, що в кожному випадку відбувається
автоматично, а що ти робиш вручну.

### Завдання 4.4. Досліди bytecode

Переконайся, що `.class`-файл існує:

```bash
find target/classes -name 'HelloDeveloper.class'
```

Переглянь структуру скомпільованого класу:

```bash
javap -classpath target/classes daniil.dumshenko.previousLessons.HelloDeveloper
```

За бажанням переглянь інструкції bytecode:

```bash
javap -c -classpath target/classes daniil.dumshenko.previousLessons.HelloDeveloper
```

Не потрібно запам'ятовувати bytecode. Треба зрозуміти, що JVM виконує не
вихідний `.java`-файл безпосередньо.

Відповідай у конспекті:

1. Який файл є source code?
2. Який файл містить bytecode?
3. Яка команда компілює source code?
4. Яка команда запускає клас у JVM?
5. Чим Maven допоміг порівняно з ручним `javac`?

### Завдання 4.5. Перевір Git-стан

Виконай:

```bash
git status
git diff
git diff --stat
```

Зверни увагу: у репозиторії можуть уже бути зміни, які не стосуються Дня 1.
Не видаляй і не перезаписуй їх.

Визнач:

- які файли були змінені до початку сьогоднішньої роботи;
- які файли створені або змінені сьогодні;
- які саме рядки належать до сьогоднішнього завдання.

Підготуй для commit лише файли Дня 1:

```bash
git add TASKS.md learning-journal.md src/main/java/daniil/dumshenko/HelloDeveloper.java
git diff --cached
```

Уважно переглянь staged diff. Не додавай через `git add .` сторонні або
незавершені зміни.

Рекомендоване повідомлення commit:

```text
Complete day 1 Java environment exercises
```

Сам commit виконуй лише тоді, коли свідомо готовий зберегти ці зміни:

```bash
git commit -m "Complete day 1 Java environment exercises"
```

### Критерій завершення блоку

- Програма має весь необхідний вивід.
- Вона запускається через IntelliJ IDEA, Maven і ручні команди.
- `.class`-файл знайдений і переглянутий через `javap`.
- Різниця між трьома способами запуску пояснена.
- До commit підготовлені лише файли поточного дня.

## 5. Перевірка та рефакторинг — 30 хвилин

Автоматичні unit tests для `println` сьогодні не потрібні. Перевір результат
через запуск програми та аналіз коду.

### Завдання 5.1. Функціональна перевірка

Запусти програму й перевір:

1. Заголовок присутній.
2. Ім'я студента присутнє.
3. Цільова роль указана правильно.
4. Країна — Germany.
5. День — `1 of 84`.
6. Тема дня присутня.
7. Розділювачі мають однакову довжину.
8. Немає порожніх випадкових рядків.

### Завдання 5.2. Перевірка помилок

Навмисно виконай по одному разу, а потім виправ:

- запусти неправильне ім'я класу;
- прибери одну подвійні лапки в рядку;
- зміни регістр однієї літери в назві файла або класу.

Для кожного випадку запиши:

- команду або зміну;
- головний рядок помилки;
- причину;
- виправлення.

Не залишай навмисні помилки в кінцевому коді.

### Завдання 5.3. Рефакторинг

Переглянь `HelloDeveloper.java`:

- чи зрозуміла назва класу;
- чи правильний package;
- чи вирівняні відступи;
- чи немає закоментованого непотрібного коду;
- чи немає зайвих imports;
- чи кожен рядок виводу потрібний;
- чи немає орфографічних помилок.

Потім виконай:

```bash
mvn compile
java -cp target/classes daniil.dumshenko.previousLessons.HelloDeveloper
git diff --check
```

### Критерій завершення блоку

- Пройдені всі 8 функціональних перевірок.
- Досліджені 3 типові помилки.
- Після виправлень `mvn compile` завершується успішно.
- `git diff --check` не показує whitespace errors.

### Завдання 3.3. Дві проблемні ситуації

Поясни, чому результат може відрізнятися від математичного очікування.

### Завдання 4.1. `ArrayStatistics`

Створи файл:

```text
src/main/java/daniil/dumshenko/ArrayStatistics.java
```

Програма повинна працювати з масивом:

```java
int[] transactions = {120, -45, 300, -80, 50, -20, 700};
```

Реалізуй без готових методів для статистики:

1. `sum` усіх елементів;
2. `min`;
3. `max`;
4. `average`;
5. кількість додатних значень;
6. кількість від'ємних значень;
7. окрему суму доходів;
8. окрему суму витрат;
9. копію масиву у зворотному порядку;
10. вивід оригінального та розвернутого масивів.

Важливо:

- `average` має бути `double`, не `int`;
- не використовуй `Collections`, streams або готовий reverse;
- для виводу масивів використовуй `Arrays.toString`;
- назви змінних мають пояснювати зміст, а не механіку.

Очікувані ідеї для перевірки:

- у масиві є і доходи, і витрати;
- витрати можна рахувати як суму від'ємних чисел або як абсолютну суму, але
  обраний варіант треба явно підписати у виводі;
- розвернутий масив не повинен змінювати оригінальний.

### Завдання 5.1. Функціональна перевірка
